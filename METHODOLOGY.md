Comparison to Stripe's built-in analytics
The take-home asks how my SQL-calculated MRR compares to Stripe's built-in analytics. Stripe's Billing overview dashboard does not display data from test-clock subscriptions. I confirmed this directly with Stripe support, who explained: "The Billing overview dashboard and its analytics don't include data from test clock subscriptions. Test clocks exist in a separate testing environment, and their data doesn't flow into the standard dashboard metrics like MRR, active subscribers, or revenue charts."
Stripe's recommended workaround for analyzing test-clock data is to "Use the API to query your test clock subscriptions directly — you can list subscriptions filtered by test clock ID and calculate metrics programmatically." This is exactly what my pipeline does — the ETL extracts subscription and invoice data from Stripe via the API into BigQuery, and the SQL computes MRR from those raw records. The validation script (scripts/validate_mrr.py) provides an independent Python-based MRR calculation against the same data, with both implementations producing identical results per month.
In production with real-time billing data, this constraint wouldn't apply — Stripe's dashboard would correctly display historical MRR and could serve as an additional comparison reference. The methodology and pipeline architecture are designed to be source-data-driven regardless of whether dashboard comparison is available.
